Based on a visual analysis of the inspiration pack against the current Hydra V2 shell, here is a concrete plan for evolving the UI.

### 1. Exact Visual Motifs Missing from Current Hydra

*   **High-Contrast Emissive Palette:** The current dark-olive-and-amber theme is muted. It lacks the stark, functional "emissive-on-black" quality seen in the *NGE* and *Hermes* images, where information appears as glowing light on a true dark background.
*   **Textural, Layered Backgrounds:** Hydra's background is a single flat color. The inspiration uses subtle, dark-on-dark patterns (like the *NGE* hex grids or *Hermes* scanlines) to create a sense of depth and a textured canvas for the UI to sit upon.
*   **Strong Typographic Hierarchy:** The inspiration materials use two distinct typographic styles for clear hierarchy: a heavy, condensed, all-caps font for titles and headers, and a clean monospaced font for data, logs, and readouts. Hydra currently uses a single, general-purpose sans-serif font.
*   **Sharp, Geometric Framing:** Hydra uses thick outlines and some rounded corners. The inspiration's aesthetic is defined by sharp, 90-degree angles and thin, 1-pixel rule lines that create a precise, grid-like structure.
*   **Glow and Bloom Effects:** Key text and status indicators in the inspiration images have a subtle "glow" or "bloom," as if generated by a CRT display. This makes them feel active and integrated into the "screen" rather than just being flat text.

### 2. Current Hydra Elements to Remove or Simplify

*   **Thick, Rounded Panel Borders:** The main panel outlines are heavy and rounded. These should be removed and replaced with thin, sharp, 1-pixel rule lines or simple gutters to create a more technical and less "web-like" feel.
*   **Over-sized Call-to-Action Buttons:** The large, solid amber "LAUNCH TMUX SHELL" button is visually loud. This should be simplified to a more integrated, text-based, or bracketed style (e.g., `[ LAUNCH ]`) that matches the utilitarian aesthetic.
*   **Inconsistent Status Tags:** The various tags (`[TARGET]`, `ONLINE`, `MUX READY`, etc.) have different styles, colors, and casings. These should be consolidated into a single, sharp-cornered, monospaced component for visual consistency.
*   **Excessive Internal Padding:** Panels and list items have generous padding, which lowers information density. This should be reduced to pack data more tightly, reinforcing the "power-user tool" feeling.

### 3. Concrete Implementation Map for Phase 1/2

*   **Global Shell (`App.qml`):**
    *   **Surface:** Change the root background from olive-green to a near-black (`#0A0A0A`).
    *   **Texture:** Implement a faint, repeating hexagonal grid pattern over the background using a `ShaderEffect` or tiled `Image` with low opacity.
*   **Panels (Left "Launch" & Right "Sessions"):**
    *   **Zoning:** Remove the thick borders. Define panels with a slightly lighter charcoal background (`#1A1A1A`) sitting on top of the global hex-grid background, creating depth. Use thin, 1px accent-colored lines for internal divisions.
*   **Headers ("HYDRA", "SESSIONS", "TARGET"):**
    *   **Typography:** Change the font to a bold, condensed, all-caps style.
    *   **Style:** Apply a subtle text glow effect using the primary UI accent color.
*   **Data Lists (Repositories, Worktrees, Active Sessions):**
    *   **Typography:** Change all file paths, commit hashes, and other technical readouts to a monospaced font.
    *   **Shape:** Remove rounded corners on list items. Use 1px horizontal lines as separators.
    *   **State:** Use a solid accent color fill for the *selected* item's background.
*   **Status Readouts & Tags ("MUX READY", "[TARGET]", "ONLINE"):**
    *   **Component:** Create a single, standardized QML component for all status tags.
    *   **Style:** It should use a sharp-cornered rectangle, a monospaced font, and consistent color-coding (e.g., Green for Ready/Online, Amber for Idle/Standby).

### 4. Strict List of No More Than 12 Visual Directives

1.  Adopt a high-contrast, emissive-on-black color palette.
2.  Replace the olive theme with a charcoal/black base and one vibrant accent color (cyan or red-orange).
3.  Eliminate all rounded corners; enforce sharp, 90-degree angles on all UI elements.
4.  Simplify panel and list-item borders to thin, 1-pixel rule lines.
5.  Introduce a subtle, dark background texture (e.g., a hex grid) to add depth.
6.  Use a bold, condensed, all-caps font for all panel and section headers.
7.  Use a clean, monospaced font for all data readouts, file paths, and technical info.
8.  Apply a subtle glow/bloom effect to primary headers and active state indicators.
9.  Increase information density by reducing excessive padding within panels.
10. Standardize all status tags into a single, sharp, monospaced component.
11. Redesign primary buttons to be text-based or bracketed `[ACTION]` style.
12. Use color functionally for status: Green for "ready," Amber for "pending," Red for "alert."
